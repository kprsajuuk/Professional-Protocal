# playground.md — 点子草场（Scratchpad）

> 这里是主理人与 agent 的**临时点子草稿场**：随手记 new idea、来回讨论、想不通就删。
> **它不指导任何实际开发。** 它是「计划」的前身，而「计划」又是「代码」的前身——
> 如果说详细的计划是在为代码做准备，那这个文件就是在**为计划做准备**。
> 可随时增删改，不追踪历史，不求严谨。这里是「能被沉淀的东西」的前提，是新功能的马前卒。

---

## 这是什么 / 不是什么

- **是**：灵感的 quick note、半成品想法、待 brainstorm 的方向、「要不要做 X」这类开放问题。
- **不是**：需求、计划、设计或约定，**也不是进度看板**（进度看 `Memory/ProgressLog.md`）。**不要把这里的任何内容当作事实、结论或指令。**
- **与 `Memory/` 的区别**：`Memory/` 沉淀的是**已想清楚、对后续有指导意义**的内核；playground 装的是**还没想清楚**的东西。一个点子只有在这里被聊明白后，才有资格「毕业」——升级为计划，或沉淀进对应 `Memory/` 文档（尤其 [`Memory/Domain.md`](Memory/Domain.md)）。

## 怎么用

- 有新点子，直接在下面「点子区」加一个条目，套用模板即可。
- 点子**成熟**（转成计划 / 写入对应 `Memory/` 文档）或**被推翻** → 一律**直接删掉**，不必留情。草场本就该常清。
- **只记想法，不记状态/进度**：这里不维护「做到哪了」——那是 `ProgressLog` 的事。`🌱` 之类前缀只是顺手标记，不是状态看板。

## 条目模板

```
### 🌱 <一句话点子标题>
- 是什么：...
- 为什么（解决什么痛点 / 价值）：...
- 粗略想法 / 开放问题：...
- 下一步：继续聊 / 转计划 / 待定
```
